### ISSUE 3 — IEM Cache Has No TTL (ROOT CAUSE OF ALL LOSING TRADES)
**Severity: P1 — Strategy Correctness**

The `sync.Map` cache in `weather_iem.go` stores IEM resolver results with **no expiry mechanism**. During a long-running session spanning midnight, early-morning results (e.g., Istanbul 9°C at 22:05 UTC = 01:05 Istanbul local) stayed cached for 12+ hours. By 10:46 UTC on April 3, when Istanbul was rising toward its actual 17°C+ high, the bot returned the stale "No" and placed a losing trade at 98% confidence.

Same mechanism caused Taipei losses: running high of 21°C cached correctly at 02:17 UTC, but the actual final high of 25°C wasn't reflected until hours later.

**Fix:** Add TTL to `CachedResult`:
```go
type CachedResult struct {
    Outcome    string
    Confidence float64
    CachedAt   time.Time  // ADD THIS
}
// In cache check:
if time.Since(result.CachedAt) > 45*time.Minute { invalidate }
```
